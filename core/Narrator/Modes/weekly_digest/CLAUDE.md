# 週報摘要 Mode

## Mode 定義表

| 項目 | 說明 |
|------|------|
| **Mode name** | weekly_digest（週報摘要） |
| **Purpose and audience** | 為公衛專業人員提供每週全球傳染病威脅摘要 |
| **Source layers** | ecdc_cdtr, who_disease_outbreak_news, us_cdc_han, tw_cdc_alerts |
| **Automation ratio** | 90% — 自動彙整各 Layer 資料，趨勢分析需人工確認 |
| **Content risk** | 資訊過時風險、綜合判斷偏誤 |
| **Reviewer persona** | 資料可信度審核員、邏輯一致性審核員 |

---

## 資料來源定義

本 Mode 讀取以下 Layer 的萃取結果：

| Layer | 路徑 | 優先級 |
|-------|------|--------|
| ecdc_cdtr | `docs/Extractor/ecdc_cdtr/**/*.md` | 高（歐洲監測） |
| who_disease_outbreak_news | `docs/Extractor/who_disease_outbreak_news/**/*.md` | 高（全球疫情） |
| us_cdc_han | `docs/Extractor/us_cdc_han/**/*.md` | 中（美國警報） |
| tw_cdc_alerts | `docs/Extractor/tw_cdc_alerts/**/*.md` | 中（台灣警報） |

### 時間範圍

- 預設：最近 7 天的資料
- 可指定：`「產出 2026-W05 週報」`

### 歷史報告（用於比較）

產出報告時，**必須讀取上一期報告**以進行比較分析：

```bash
# 取得上一期報告路徑
source lib/report.sh
prev_report=$(report_find_previous weekly_digest 2026-W05)
```

歷史查詢工具：

| 函式 | 用途 | 範例 |
|------|------|------|
| `report_find_previous` | 取得上一期報告 | `report_find_previous weekly_digest 2026-W05` |
| `report_list_all` | 列出所有歷史報告 | `report_list_all weekly_digest` |
| `report_search` | 搜尋報告內容 | `report_search weekly_digest "立百病毒"` |
| `report_get_range` | 取得指定週數範圍的報告 | `report_get_range weekly_digest 2026-W01 2026-W04` |

> **注意**：若上一期報告不存在（如首次產出），則跳過比較章節。

## 產出規則

### 優先級判定

依以下順序決定報告內容優先級：

1. **緊急** — CDC HAN alerts、WHO PHEIC、ECDC 高風險評估
2. **重要** — 新疫情爆發、跨國傳播、境外移入案例
3. **監測** — 持續疫情更新、例行監測數據
4. **資訊** — 政策公告、指引更新

### 去重邏輯

- 同一疫情在多個來源出現時，優先使用 WHO > ECDC > CDC 的順序
- 合併相同疾病的多筆更新，保留最新資訊
- 標註資料來源以便交叉驗證

### 歷史比較邏輯

產出「與上週比較」章節時，依以下邏輯判定：

| 分類 | 判定條件 |
|------|----------|
| **新增威脅** | 本週「本週重點」或「新興威脅」中出現，但上週報告未提及的疾病 |
| **升級警示** | 同一疾病的風險評估上升、病例數增加 >20%、或傳播範圍擴大 |
| **持續關注** | 連續 2 週以上出現在「本週重點」或「新興威脅」的疾病 |
| **已解除/降級** | 上週在「本週重點」但本週未列入，或風險評估下降 |

> **若無上一期報告**：省略整個「與上週比較」章節，不要產出空白章節。

### 語言

- 主要輸出語言：繁體中文
- 疾病名稱：中英對照

## 自我審核 Checklist

報告發布前必須確認：

- [ ] 已包含免責聲明
- [ ] 引用的資料確實存在於 `docs/Extractor/`
- [ ] 未產出無來源依據的判斷
- [ ] 推測與事實已明確區分（使用「可能」、「預計」等詞）
- [ ] 區域劃分正確（疫情歸屬正確區域）
- [ ] 時間範圍正確（僅包含本週資料）
- [ ] 已列出所有資料來源
- [ ] 若有上一期報告，已正確產出「與上週比較」章節
- [ ] 比較章節中的分類（新增/升級/持續/解除）判定正確
