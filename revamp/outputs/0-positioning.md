# 品牌定位文件

> **審查狀態**：✅ 通過（2026-02-19）

## 1. 核心資訊

| 項目 | 內容 |
|------|------|
| **網站名稱** | EpiAlert 疫情快訊 |
| **一句話描述** | 全球傳染病情報自動收集與分析系統 |
| **核心價值主張** | 運用 AI 技術自動彙整全球主要公衛機構（WHO、CDC、ECDC、UKHSA、台灣 CDC）的疫情資訊，提供即時、結構化的傳染病威脅監測與週報分析，讓關心公共衛生的民眾、專業人士和決策者能在單一平台快速掌握全球疫情動態。 |

## 2. 目標受眾

### 主要受眾 (Primary)

| 項目 | 描述 |
|------|------|
| **是誰** | 關注公共衛生議題的一般民眾、家長、計劃出國旅遊者 **[假設：基於疫情新聞關注度推測，待 Google Analytics 受眾數據驗證]** |
| **目標/需求** | 快速了解當前全球及台灣的主要傳染病威脅（如麻疹、百日咳、禽流感），評估自身及家人健康風險，做出知情決策（如是否接種疫苗、是否調整旅遊計劃） **[假設：基於疫情期間使用者行為推測]** |
| **痛點** | 官方公衛機構資訊分散於不同網站（WHO、各國 CDC），需要逐一瀏覽；資訊量龐大，難以快速抓取重點；無法一站式比對不同國家/地區的疫情趨勢 **[假設：基於作者個人經驗推測，待使用者訪談驗證]** |
| **使用場景** | 看到新聞報導某疾病疫情時想查證詳情；計劃出國前查詢目的地疫情風險；孩童出現發燒咳嗽症狀時想了解社區流行病；週末閱讀週報掌握整體疫情動態 **[假設：基於典型使用者行為推測，待 Session Recording 或使用者調查驗證]** |

### 次要受眾 (Secondary)

| 項目 | 描述 |
|------|------|
| **是誰** | 醫療專業人員（醫師、護理師、公衛人員）、研究人員、政策制定者、媒體工作者 **[假設：基於內容專業性推測，待實際訪客職業分布驗證]** |
| **目標/需求** | 監測特定疾病的全球流行趨勢、追蹤歷史資料以進行比較分析、快速獲取多國官方數據以支持專業判斷或政策建議 **[假設：基於專業工作流程推測]** |
| **痛點** | 需要跨國比較疫情數據時缺乏整合平台；歷史資料查詢耗時；缺乏語意搜尋功能難以快速找到相關案例 **[假設：基於作者與公衛專業人士非正式交流推測，待正式訪談驗證]** |
| **使用場景** | 撰寫疫情分析報告或新聞稿時查詢多國數據；臨床診斷時參考當前流行病學趨勢；制定防疫政策前了解國際最新動態；研究特定疾病的歷史爆發模式 **[假設：基於典型專業工作流程推測，待使用者調查驗證]** |

## 3. 差異化定位

| 項目 | 內容 |
|------|------|
| **與競品的差異** | **自動化 + AI 驅動**：不同於人工彙整的疫情新聞網站或單一機構官網，EpiAlert 透過自動化程式持續擷取 7 大官方來源（WHO、US CDC、ECDC、UKHSA、Taiwan CDC），由 AI（Claude Opus）進行跨來源綜合分析、趨勢判斷與歷史比較，每週自動產出結構化週報。<br>**語意搜尋 + 歷史脈絡**：整合 Qdrant 向量資料庫，提供語意搜尋功能，能快速找到相似歷史案例，補足官方網站缺乏的歷史追溯能力。<br>**一站式多國監測**：同時涵蓋北美（美國）、歐洲（ECDC、英國）、亞洲（台灣）及全球（WHO）資料，讓使用者無需跨多個網站查詢。 |
| **獨特優勢** | **即時性 + 結構化**：自動化擷取確保資料即時更新，AI 萃取將非結構化的官方公告轉為結構化內容（分類為 outbreak、alert、surveillance、imported_case 等），方便快速瀏覽。<br>**SEO/AEO 優化**：完整實作 Schema.org、SGE 標記及 YMYL 醫療免責聲明，符合搜尋引擎與 AI 助理的資訊擷取標準。<br>**開源透明**：程式碼公開於 GitHub，使用者可追溯資料來源與處理邏輯。 |
| **不做什麼** | **不提供醫療建議或診斷**：明確標註所有內容僅供參考，不構成醫療建議或專業診斷。<br>**不處理非官方來源**：僅彙整 WHO、CDC 等公信力機構的資料，不收錄社群媒體、個人部落格或未經證實的疫情傳聞。<br>**不提供即時疫情地圖**：專注於文字情報的結構化分析，不提供視覺化地圖或互動式圖表。<br>**不做疫情預測**：僅報告已發生事件與官方監測數據，不進行疫情走向預測或流行病學建模。 |

## 4. 品牌調性

| 項目 | 描述 |
|------|------|
| **語氣** | **專業中立、資訊導向**：使用官方用語，保持客觀陳述；**清晰易懂**：結構化呈現，避免專業術語過度堆砌；**謹慎負責**：每頁顯著標示免責聲明與最後審核日期 |
| **視覺風格** | **簡約實用**：基於 Just the Docs 主題的乾淨排版；**結構化**：大量使用表格、清單、分層標題；**符合 YMYL 標準**：重要資訊以視覺區塊強調 |
| **關鍵字** | 自動化（Automated）、即時（Real-time）、結構化（Structured）、多國整合（Multi-source）、AI 驅動（AI-powered） |

## 5. 成功指標

| 指標 | 目標值 | 測量方式 | 測量工具與範圍 |
|------|--------|----------|----------------|
| **週報更新頻率** | 每週一次 | 檢查檔案產出時間戳 | **工具**：`git log --since="7 days ago" --grep="weekly_digest"` 或檔案系統 `stat` 指令<br>**測量範圍**：`docs/Narrator/weekly_digest/` 目錄下最新 `.md` 檔案的 `mtime`<br>**測量頻率**：每週一執行檢查<br>**閾值**：若 7 天內無更新則觸發警報 |
| **資料來源更新成功率** | ≥ 95% **[假設]** | 統計各 Layer 的 JSONL 行數 | **工具**：GitHub Actions workflow 執行後產生的統計報告<br>**測量範圍**：`docs/Extractor/*/raw/*.jsonl` 所有 7 個 Layer<br>**計算公式**：成功 Layers 數 ÷ 總 Layers 數 × 100%<br>**測量頻率**：每次 `git push` 後自動執行<br>**成功定義**：JSONL 行數 > 0 且 fetch.sh 回傳 exit code 0 |
| **自然搜尋流量（SEO）** | 改版後 3 個月內成長 50% **[假設]** | Google Analytics / Search Console | **工具**：Google Analytics 4 (GA4) + Google Search Console (GSC)<br>**測量範圍**：GA4 報表 > 流量開發 > 自然搜尋流量<br>**基準值**：2026-02-19 前 30 天平均日自然搜尋流量<br>**測量頻率**：每週一檢視，每月產出趨勢報告 |
| **核心關鍵字排名** | 進入前 3 頁（排名 1-30） **[假設]** | Search Console | **工具**：Google Search Console > 成效報告<br>**測量範圍**：台灣地區搜尋查詢<br>**目標關鍵字**：麻疹疫情、百日咳台灣、禽流感 H5N1、Marburg 病毒、ECDC 週報、WHO 疫情通報 |
| **使用者停留時間** | 平均 ≥ 2 分鐘 **[假設]** | Google Analytics | **工具**：GA4 > 報表 > 參與 > 網頁和畫面<br>**測量範圍**：平均工作階段持續時間 |
| **連結錯誤率** | ≤ 1% | lychee 連結檢查報告 | **工具**：lychee (`.github/workflows/check-links.yml`)<br>**測量範圍**：`docs/**/*.md` 所有連結 |
| **SEO/AEO 合規率** | 100% | 內部檢查清單 | **工具**：`seo/review/CLAUDE.md` 檢查清單<br>**通過標準**：10/10 項目全數通過 |
