## 競品詳細分析

### 競品 A：ProMED (Program for Monitoring Emerging Diseases)

#### 基本資訊

| 項目 | 數據 |
|------|------|
| **URL** | https://www.promedmail.org |
| **定位** | 全球最大的傳染病早期預警系統，由人類專家策展的疫情快報平台 |
| **目標受眾** | 公衛專業人員、研究人員、政策制定者、醫療從業者、國際組織（如 WHO、CDC） |
| **成立年份** | 1994 年（32 年歷史） |
| **營運模式** | 非營利，由 ISID (International Society for Infectious Diseases) 營運 |

#### 流量數據（估算）

| 指標 | 數據 |
|------|------|
| **訂閱戶數** | 37,000+ 訂閱者，遍及 150+ 國家 **[來源：ProMED 官網]** |
| **主要流量來源** | Email 訂閱（主要）、直接流量、搜尋引擎 |
| **更新頻率** | 每日多次（視全球疫情事件而定） |

#### 技術表現

| 指標 | 分數 | 評價 | 說明 |
|------|------|------|------|
| **PageSpeed Mobile** | 47 | ❌ 差 | 遠低於我們的 70 分 |
| **SEO 分數** | 100 | ✅ 優秀 | 與我們並列 |
| **Accessibility** | 93 | ✅ 良好 | 略低於我們的 100 分 |
| **LCP** | 4.7s | ⚠️ 需改善 | 我們的 1.1s 明顯優於競品 |
| **CLS** | 0.02 | ✅ 良好 | 略高於我們的 0（完美） |
| **FCP** | 2.0s | ✅ 良好 | 我們的 1.1s 更快 |
| **TBT** | 3,460ms | ⚠️ 需改善 | 我們的 9,930ms 更差 |

#### 內容策略

| 項目 | 觀察 |
|------|------|
| **資訊架構** | 首頁 → Weekly Pulse 快報列表 → 個別報告詳情 |
| **主要內容類型** | 1. 專家策展的疫情快報（核心）<br>2. AI 生成摘要（輔助）<br>3. 歷史資料檔案（31 年）<br>4. 多語言內容（英、俄、法、西、中等） |
| **更新頻率** | 每日多次更新，依全球疫情事件即時發布 |
| **特色內容** | 1. **Weekly Pulse 時間軸**：滾動式快報列表<br>2. **專家評論**：人類專家對疫情的專業分析<br>3. **多網絡支援**：ProMED AMR、MBDS、Middle East 等子網絡<br>4. **31 年歷史資料庫**：可回溯至 1994 年 |
| **分類方式** | 按疾病、地點、物種、網絡分類 |

#### 優勢與特色

1. **權威性與信任度**：32 年品牌歷史，被 WHO、CDC 等官方機構信賴引用，ProMED 準確率達 91%（高於 Google News 的 81%）
2. **專家策展**：不同於純 AI 系統，由傳染病專家人工審核、評論、驗證
3. **全球網絡**：37,000+ 訂閱者遍及 150+ 國家，形成全球情報網
4. **多語言支援**：提供英、俄、法、西、中等多種語言版本
5. **Email 訂閱機制**：主動推播給訂閱者，黏著度高
6. **歷史資料完整**：31 年疫情檔案，研究價值高
7. **整合性**：與 HealthMap、EIOS (WHO) 整合，數據互通

#### 可改善之處

1. **技術效能差**：PageSpeed 僅 47 分，LCP 4.7 秒（我們僅 1.1 秒）
2. **視覺設計老舊**：首頁採用傳統表格列表，缺乏視覺化元素
3. **行動體驗不佳**：未優化行動版，載入速度慢
4. **搜尋功能不明顯**：雖有搜尋功能，但 UI 不突出
5. **無結構化週報**：僅有即時快報，缺乏週報整合分析（這是我們的優勢）

| **TBT** | 3,750ms | ⚠️ 需改善 | 我們的 9,930ms 更差 |

#### 內容策略

| 項目 | 觀察 |
|------|------|
| **資訊架構** | 互動地圖（主要介面） → 側邊欄搜尋/篩選 → 個別疫情詳情 |
| **主要內容類型** | 1. **互動式全球地圖**（Google Maps 整合）<br>2. 疫情標記（國家級與地方級）<br>3. 疾病分類（12 大類：呼吸道、腸胃道、出血性、神經、病媒、皮疹等）<br>4. 預設搜尋（如「Coronavirus Surveillance」、「Avian Influenza」） |
| **更新頻率** | 即時更新（AI 自動化，每 15 分鐘掃描一次） |
| **特色內容** | 1. **Outbreaks Near Me**：使用地理定位顯示附近疫情<br>2. **顏色編碼**：依疫情嚴重度分級（低→高）<br>3. **多資料源整合**：Google News、ProMED、WHO、官方公告<br>4. **儲存搜尋**：登入使用者可儲存搜尋條件並接收 Email 通知 |
| **分類方式** | 按疾病、地點、來源、物種、日期範圍篩選 |

#### 優勢與特色

1. **視覺化優勢**：全球唯一提供即時互動地圖的疫情監測平台，直觀易懂
2. **即時性**：每 15 分鐘掃描一次，速度快於多數官方機構
3. **自動化**：使用 AI/NLP 技術自動分類、定位、標記疫情資料
4. **多資料源整合**：聚合 Google News、ProMED、WHO、官方公告等數千個來源
5. **地理定位功能**：Outbreaks Near Me 自動顯示使用者附近的疫情
6. **行動應用**：提供 iOS/Android App（Outbreaks Near Me）
7. **學術背景**：哈佛醫學院、波士頓兒童醫院支持，學術可信度高

#### 可改善之處

1. **技術效能極差**：PageSpeed 僅 31 分（所有競品中最低），LCP 7.4 秒
2. **需要 JavaScript**：無 JS 則完全無法使用，對 SEO 和 Accessibility 不利
3. **資訊密度低**：地圖視覺化雖直觀，但不適合快速瀏覽大量文字資訊
4. **無週報整合**：僅提供即時疫情點，缺乏週報式綜合分析
5. **無歷史追溯**：雖有日期篩選，但無語意搜尋或歷史脈絡分析

| **目標受眾** | 1. 各國政府與衛生部門<br>2. 國際公衛組織<br>3. 醫療專業人員<br>4. 媒體與研究人員 |
| **成立年份** | 1990 年代（歷史資料可回溯至 1990 年） |

| 指標 | 評價 | 說明 |
|------|------|------|
| **設計** | ✅ 現代化 | Material Design Components (MDC) 框架 |
| **多語言** | ✅ 優秀 | 7 種語言（中、英、法、俄、西、阿、葡） |
| **篩選功能** | ✅ 良好 | 按年份篩選（1990-2026） |
| **資訊架構** | ✅ 清晰 | 卡片式布局，響應式設計 |

#### 內容策略

| 項目 | 觀察 |
|------|------|
| **資訊架構** | Mega-menu（Health Topics、Countries、Newsroom、Emergencies、Data、About WHO） → DON 列表 → 個別報告 |
| **主要內容類型** | 1. DON 文章（急性疫情通報）<br>2. Situation Reports（持續事件追蹤）<br>3. Rapid Risk Assessments（風險評估）<br>4. Fact Sheets / Q&As（公眾教育） |
| **更新頻率** | 不定期（依事件嚴重度，重大疫情每日更新） |
| **特色內容** | 1. **官方權威**：各國政府須依 IHR 通報 WHO<br>2. **多語言**：7 種語言同步發布<br>3. **地區細分**：AFRO、PAHO、EMRO、EURO、SEARO、WPRO 六大區域<br>4. **數據儀表板**：COVID-19 Dashboard 等視覺化工具 |

#### 優勢與特色

1. **最高權威性**：全球公衛標準，各國政府與國際組織的主要參考來源
2. **官方驗證**：所有資料經 WHO 專家團隊驗證
3. **多語言覆蓋**：7 種語言同步，全球通用
4. **政策影響力**：DON 發布可直接影響各國防疫政策、旅遊警示、疫苗政策
5. **歷史完整**：36 年歷史資料，研究價值極高

#### 可改善之處

1. **更新速度慢**：官僚流程導致發布速度慢於 ProMED、HealthMap 等民間平台
2. **無自動化整合**：各 DON 文章獨立，缺乏跨來源綜合分析（這是我們的優勢）
3. **資訊密度低**：單篇文章詳細但難以快速瀏覽整體疫情
4. **無訂閱通知**：未提供 Email 訂閱或 RSS（僅有 RSS，但不明顯）
5. **搜尋功能弱**：站內搜尋功能不佳，難以找到歷史案例

5. **專家支援**：提供客製化培訓與技術支援

1. **付費門檻高**：非開放平台，需機構訂閱（一般民眾無法使用）
2. **介面複雜**：功能強大但學習曲線陡峭
3. **無即時疫情**：專注歷史資料與診斷，缺乏即時疫情快報
4. **無社群功能**：單向資訊提供，缺乏使用者互動

### 1. **定位：AI 驅動的「週報式情報整合平台」**

- 與 ProMED、HealthMap 的「即時快報」區隔

- 與 GIDEON、BlueDot 的「專業付費平台」區隔（我們是免費開放）

**如何強化**：
- 首頁標語改為：「AI 驅動的全球疫情週報 — 7 大官方來源一站整合」
- 強調「週報」（Weekly Digest）而非「即時快報」（Real-time Alerts）
- 在首頁加入「與其他平台的差異」對照表

### 2. **技術：SEO/AEO 優化 + 語意搜尋**

**說明**：
- 我們的 SEO/AEO 優化（Schema、SGE、YMYL）超越所有競品
- 語意搜尋（Qdrant）是獨特優勢，競品均無
