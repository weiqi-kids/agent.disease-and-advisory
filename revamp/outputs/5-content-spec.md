| 維度 | 規格 | 範例 |
|------|------|------|
| 正式度 | 中性偏正式 | 使用「您」而非「你」，但避免過度艱澀 |

| 使用 | 避免 |
|------|------|

| 高風險、中風險、低風險 | 嚴重、危險、恐怖 |
| 建議接種疫苗 | 必須接種疫苗 |
| 根據 WHO/CDC 資料 | 據報導、據悉 |
| 境外移入、imported case | 外國傳入 |
| 萃取、彙整 | 抄襲、複製 |

## 1. 頁面目標

### 主要目標

讓新訪客在 10 秒內理解網站價值（AI 驅動週報 + 7 大官方來源整合），並引導點擊查看完整週報。

### 次要目標

1. 讓回訪者快速掌握本週最重要疫情（5-8 筆，含風險等級）
2. 建立信任感（展示官方來源、開源透明）
3. 引導使用者訂閱/搜尋/深入瀏覽

### 成功指標

| 指標 | 目標值 | 測量方式 |
|------|--------|----------|
| 跳出率 | < 50% | GA4 |
| 主要 CTA 點擊率 | > 15% | GA4 事件追蹤 |
| 平均停留時間 | > 1 分鐘 | GA4 |
| 首次內容繪製 (FCP) | < 1.5 秒 | PageSpeed Insights |

##### 表格格式

| 風險 | 疾病 | 地區 | 摘要 | 連結 |
|------|------|------|------|------|
| 高風險 | Marburg 病毒 | Rwanda | 累計 66 例確診，15 例死亡 | 詳情 |
| 中風險 | 麻疹 | 日本 | 東京都疫情持續，累計 120+ 例 | 詳情 |
| 中風險 | 百日咳 | 台灣 | 本週新增 15 例，累計 89 例 | 詳情 |
| 低風險 | 禽流感 H5N1 | 美國 | 乳牛群散發案例，無人傳人 | 詳情 |

##### 風險等級標記定義

| 風險等級 | 標記 | CSS class | 判定標準 |
|----------|------|-----------|----------|
| 高風險 | 紅底白字 | `.risk-high` | PHEIC、案例數 >1000、死亡數 >100、跨國傳播且致死率 >5% |
| 中風險 | 黃底黑字 | `.risk-medium` | 區域性爆發、案例數 100-1000、新興疾病或變異株 |
| 低風險 | 綠底黑字 | `.risk-low` | 散發案例 <100、常規監測、無明顯擴散 |

```markdown
## 資料來源與處理流程

### 資料來源

我們僅彙整以下 7 個官方公衛機構的公開資訊：

| 來源 | 機構 | 說明 |
|------|------|------|
| WHO DON | 世界衛生組織 | 疾病爆發新聞 (Disease Outbreak News) |
| US CDC HAN | 美國疾病管制中心 | 健康警報網絡 (Health Alert Network) |
| US CDC MMWR | 美國疾病管制中心 | 發病率與死亡率週報 |
| US Travel Health | 美國疾病管制中心 | 旅遊健康通知 |
| ECDC CDTR | 歐洲疾病預防控制中心 | 傳染病威脅報告 |
| UKHSA Updates | 英國健康安全局 | 疫情更新 |
| Taiwan CDC | 台灣疾病管制署 | 疫情警報 |

### 處理流程

1. **自動擷取**：每週自動從官方 API/RSS 擷取最新資料
2. **AI 萃取**：使用 Claude Sonnet 將非結構化公告轉為結構化內容（分類、標籤、摘要）
3. **向量化**：將萃取結果存入 Qdrant 向量資料庫，支援語意搜尋
4. **週報產出**：使用 Claude Opus 進行跨來源綜合分析，產出週報

### AI 模型說明

- **Claude Sonnet**：用於單一來源的萃取與結構化，效率高、成本低
- **Claude Opus**：用於週報產出，需要跨來源綜合分析、趨勢判斷
- **OpenAI Embeddings**：用於向量化，支援語意搜尋
```

# 疾病專頁模板 (/disease/*)

## 基本資訊

| 項目 | 內容 |
|------|------|
| 頁面 URL | `/disease/{disease-name}` |
| 所屬階段 | Phase 2 |
| 優先級 | P1 |
| 來源 | 策略計劃 4.5 |

## 頁面目標

整合特定疾病的所有歷史記錄，提供疾病背景資訊與趨勢分析。

## 內容大綱

1. **疾病概述**：名稱、別名、病原體、症狀、傳播途徑、潛伏期
2. **預防與治療**：疫苗、預防措施、治療方式
3. **最新動態**：近 30 天相關記錄（3-5 筆）
4. **趨勢分析**：過去 12 個月折線圖、地理分布
5. **歷史記錄表格**：日期、來源、事件類型、地區、摘要（每頁 20 筆）
6. **相關疾病**：相似疾病連結
7. **外部資源**：WHO、CDC 疾病頁面連結

# sitemap.xml

## 基本資訊

| 項目 | 內容 |
|------|------|
| 檔案位置 | `/sitemap.xml` |
| 所屬階段 | Phase 1 |
| 優先級 | P0 |
| 來源 | 策略計劃 S1、差距分析 G3 |

## 規格

### 實作方式

使用 Jekyll sitemap 外掛自動生成：

```yaml
# _config.yml
plugins:
  - jekyll-sitemap

# 排除不需要索引的頁面
defaults:
  - scope:
      path: "Extractor/*/raw/*"
    values:
      sitemap: false
  - scope:
      path: "*.jsonl"
    values:
      sitemap: false
```

### 預期結構

```xml
<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <url>
    <loc>https://epialert.com/</loc>
    <lastmod>2026-02-19</lastmod>
    <changefreq>daily</changefreq>
    <priority>1.0</priority>
  </url>
  <url>
    <loc>https://epialert.com/about</loc>
    <lastmod>2026-02-19</lastmod>
    <changefreq>monthly</changefreq>
    <priority>0.8</priority>
  </url>
  <url>
    <loc>https://epialert.com/Narrator/weekly_digest/2026-W08-weekly-digest</loc>
    <lastmod>2026-02-17</lastmod>
    <changefreq>weekly</changefreq>
    <priority>0.9</priority>
  </url>
  <!-- 2,450+ 頁面... -->
</urlset>
```
